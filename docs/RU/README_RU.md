# Git-Brandmauer

**Git-Brandmauer** — это **модульный фреймворк для безопасного управления репозиториями Git и контроля сети**.

Он предоставляет:

- **Интерактивные меню** для управления репозиториями, установки режимов репозитория (SAFE, NORMAL, OPEN) и выполнения сетевых операций.

- **Режимы репозитория с учетом состояния**, хранящиеся в `~/.git-security/state/`, гарантирующие автоматическую блокировку таких операций, как `git push`, в ограниченных режимах.

- **Умная установка хуков Git**, при этом `smart-install-hooks.sh` обновляет только измененные хуки и сохраняет изменения пользователя.

- **Скрипты управления сетью** (`net/`) для проверки состояния сети, приостановки соединений или запуска аварийных завершений работы.

- **Система логирования** в `~/.local/share/brandmauer/logs/` для отслеживания событий безопасности, сетевых действий и применения хуков.

- **Модульная структура каталогов**:

- `LIB_DIR` — статические скрипты и библиотеки (`/usr/local/lib/brandmauer`)

- `SECURITY_ROOT` — локальное состояние, хуки (`~/.git-security`)

- `LOG_DIR` — журналы (`~/.local/share/brandmauer/logs`)

- `NET_DIR` — скрипты для работы с сетью
- **Улучшенное обеспечение безопасности** с помощью хуков (pre-push, pre-commit), которые учитывают режимы репозитория и регистрируют все действия.

Git-Brandmauer теперь представляет собой **полноценную систему**, а не просто набор скриптов — он объединяет безопасность репозитория, управление сетью, автоматизацию хуков и логирование в единой модульной структуре.


## Структура системы (версия 1.0, стабильная)

### Обзор

brandmauer — это инструмент безопасности системного уровня, разработанный для сред на основе Ubuntu.
Структура файловой системы соответствует принципам, совместимым со стандартом иерархии файловой системы и типичными соглашениями Ubuntu (экосистема от Canonical Ltd.).

В версии 1.0 определена **стабильная и фиксированная структура каталогов**.
Эта структура не изменится в будущих минорных релизах.

## Пути установки

### 1. Точка входа CLI
```bash
/usr/local/bin/brandmauer
```

- Единственная исполняемая точка входа
- Выступает в качестве диспетчера
- Не содержит бизнес-логики

### 2. Основная системная библиотека
```bash
/usr/local/lib/brandmauer/
```

Внутренняя структура:
```bash
lib/brandmauer/
├── core/ # базовые утилиты, логирование, разделяемые функции
├── git/ # подсистема git-brandmauer
├── net/ # подсистема сетевой безопасности
├── hooks/ # шаблоны хуков git
├── ui/ # меню, форматирование вывода, цвета
└── config.sh # глобальные настройки Конфигурация
```

### Гарантия стабильности (v1.0)
Стабильными считаются следующие:
Название каталога brandmauer
Расположение `/usr/local/lib/brandmauer`
Подкаталоги:
* `core`
* `git`
* `net`
* `hooks`
* `ui`
В будущем могут быть добавлены модули, но существующие пути не изменятся.

### 3. Общие данные (необязательно)
```bash
/usr/local/share/brandmauer/
```

Зарезервировано для:
* шаблонов
* статических ресурсов
* фрагментов документации
Этот каталог может не существовать, если он не используется.

### 4. Каталог состояния пользователя
```bash
$HOME/.git-security/
```

Хранилища:
* режимы репозитория (SAFE / HARD)
* список зарегистрированных репозиториев
* состояние во время выполнения
Данные пользователя никогда не удаляются без подтверждения.
Это местоположение стабильно и является частью дизайна общедоступной системы.

## Архитектурная модель

Внутренний поток:
```bash
CLI → Core → Git / Net → Hooks
```

Где:
* CLI = интерфейсный слой
* Core = общая логика
* Git / Net = функциональные подсистемы
* Hooks = слой контроля репозитория

## Модель безопасности (v1.0)

### Обзор
brandmauer реализует локальный уровень управления командами на уровне репозитория для операций Git.
Его цель — снизить риск случайных или небезопасных удаленных операций в репозиториях, размещенных на GitHub и управляемых локально через Git. Система работает для каждого репозитория и обеспечивает соблюдение режимов безопасности, которые либо разрешают, либо ограничивают потенциально опасные команды.

### Основная концепция
Каждому зарегистрированному локальному репозиторию назначается режим безопасности:
`OPEN` — разрешены удаленные операции
`SAFE` — некоторые удаленные операции блокируются
Режим хранится локально и применяется автоматически.

### Контролируемые команды
В режиме SAFE следующие команды Git ограничены:
* `git push`
* `git pull`
* `git fetch`
* `git rebase`
* `git merge`
Эти команды считаются потенциально опасными, поскольку они:
* изменяют удаленную историю
* синхронизируют удаленные изменения
* перезаписывают историю коммитов
* объединяют внешнее состояние с локальными ветками

### Уровень обеспечения безопасности

Обеспечение безопасности реализуется посредством:
* логики управления на уровне репозитория
* интеграции хуков Git (где применимо)
* механизмов перехвата команд
Защита применяется к каждому зарегистрированному репозиторию, а не глобально.

### Контролируемые команды Это позволяет:
* обеспечить строгую защиту критически важных репозиториев
* обеспечить гибкую работу репозиториев разработки

## Режимы работы

### Открытый режим
* Все команды Git работают в обычном режиме.

* Ограничения не применяются.

### Безопасный режим
* Контролируемые команды блокируются.

* Пользователь получает четкое сообщение с объяснением ограничения.

* Локальные операции (добавление, фиксация, проверка статуса и т. д.) остаются без изменений.

### Область защиты
brandmauer работает:
* только на локальном компьютере
* только для зарегистрированных репозиториев
* только на уровне команд Git
Он не:
* изменяет настройки сервера GitHub
* вмешивается в аутентификацию
* заменяет сам Git
* обеспечивает соблюдение политик на удаленной стороне

### Принципы проектирования
1. Безопасность с приоритетом локальности
2. Контроль для каждого репозитория
3. Минимальное вторжение
4. Явное знание режима
5. Контроль соблюдения пользователем

### Модель угроз (практическая)
Система предназначена для предотвращения:
* случайных git push в защищенные ветки
* непреднамеренных перезаписей истории
* импульсивной удаленной синхронизации
* ошибок рабочего процесса на важных этапах разработки
Она не предназначена для защиты от:
* вредоносных атак на системном уровне
* скомпрометированных учетных записей пользователей
* нарушений на стороне сервера

## Подсистема сетевой безопасности

Модуль net является неотъемлемой частью системы и не является опциональным в версии 1.0.

Включенные инструменты:
* panic.sh
* burn-zip-archives.sh
* passwd_offline.sh
* net_check.sh
* net-pause.sh
Эти инструменты считаются основными утилитами безопасности и являются частью архитектуры системы.

## Политика обновления

Незначительные обновления (v1.x) не изменят структуру файловой системы.
Изменения в основных версиях (v2.0+) могут привести к структурным изменениям.
Скрипты установки и удаления зависят от этой структуры.

## Философия проектирования

brandmauer — это системный уровень безопасности для:
* управления репозиториями Git
* управления состоянием сети
* действий в чрезвычайных ситуациях
Он предназначен для локальных машин на базе Ubuntu и не разработан для мобильных устройств или кроссплатформенного развертывания.
